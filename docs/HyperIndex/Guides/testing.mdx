## Introduction

HyperIndex ships a purpose-built testing framework powered by `createTestIndexer()`. Write tests against the same indexer pipeline that runs in production — no database, no Docker, no manual mock wiring.

The framework integrates with [Vitest](https://vitest.dev/) and provides:

- **Three ways to feed events** — auto-exit, explicit block range, or simulate
- **Snapshot-driven assertions** — capture every entity set/delete per block
- **Direct entity access** — read and preset state between processing runs
- **Parallel test execution** — worker thread isolation for fast CI

## Learn by Doing

The Greeter template includes complete tests that demonstrate best practices:

1. Generate the `greeter` template:

```bash
pnpx envio init template -d greeter -t greeter -n greeter
```

2. Run tests:

```bash
pnpm test
```

3. Explore `src/indexer.test.ts` to see how the tests are written.

## Getting Started

### Prerequisites

- A functioning indexer setup with schema and event handlers
- Envio CLI v3.0.0-alpha.21 or above
- [Vitest](https://vitest.dev/) installed as a dev dependency

### Setup

1. Install Vitest:

```bash
pnpm add -D vitest@4.0.16
```

2. Add the test script to `package.json`:

```json
{
  "scripts": {
    "test": "vitest run"
  }
}
```

3. Generate the testing library:

```bash
pnpm envio codegen
```

This generates the `createTestIndexer` function based on your schema and configuration. Always re-run codegen when you change your schema or config.

4. Create your test file (e.g., `src/indexer.test.ts`):

```typescript
import { describe, it } from "vitest";
import { createTestIndexer } from "generated";

describe("My indexer", () => {
  it("processes events correctly", async (t) => {
    const indexer = createTestIndexer();
    // ... your test logic
  });
});
```

## Three Ways to Feed Events

### 1. Auto-Exit

Processes the first block with matching events, then exits. Each subsequent `process()` call continues where the last one stopped. Zero config needed — great for exploratory testing.

```typescript
import { describe, it } from "vitest";
import { createTestIndexer } from "generated";

describe("ERC20 indexer", () => {
  it("processes the first block with events", async (t) => {
    const indexer = createTestIndexer();

    const result = await indexer.process({ chains: { 1: {} } });

    // Auto-filled by Vitest on first run — just review and commit
    t.expect(result).toMatchInlineSnapshot(`
      {
        "changes": [
          {
            "Transfer": {
              "sets": [
                {
                  "blockNumber": 10861674,
                  "from": "0x0000000000000000000000000000000000000000",
                  "id": "1-10861674-23",
                  "to": "0x41653c7d61609D856f29355E404F310Ec4142Cfb",
                  "transactionHash": "0x4b37d2f343608457ca...",
                  "value": 1000000000000000000000000000n,
                },
              ],
            },
            "block": 10861674,
            "chainId": 1,
            "eventsProcessed": 1,
          },
        ],
      }
    `);
  });
});
```

### 2. Explicit Block Range

Pin to specific blocks for deterministic CI snapshots. Use `startBlock` and `endBlock` to define the exact range.

```typescript
it("processes a specific block range", async (t) => {
  const indexer = createTestIndexer();

  const result = await indexer.process({
    chains: {
      1: {
        startBlock: 10_861_674,
        endBlock: 10_861_674,
      },
    },
  });

  t.expect(result.changes).toHaveLength(1);
  t.expect(result.changes[0].eventsProcessed).toBe(1);
});
```

### 3. Simulate

Feed typed synthetic events for pure unit tests. No network, no block ranges — ideal for testing handler logic in isolation.

```typescript
it("creates a User entity from a NewGreeting event", async (t) => {
  const indexer = createTestIndexer();

  await indexer.process({
    chains: {
      137: {
        simulate: [
          {
            contract: "Greeter",
            event: "NewGreeting",
            params: { greeting: "Hello", user: "0x123..." },
          },
        ],
      },
    },
  });

  const user = await indexer.User.getOrThrow("0x123...");
  t.expect(user.latestGreeting).toBe("Hello");
  t.expect(user.numberOfGreetings).toBe(1);
});
```

You can pass multiple events in the `simulate` array — they're processed in order, automatically:

```typescript
await indexer.process({
  chains: {
    137: {
      simulate: [
        {
          contract: "Greeter",
          event: "NewGreeting",
          params: { greeting: "Hello", user: "0x123..." },
        },
        {
          contract: "Greeter",
          event: "NewGreeting",
          params: { greeting: "Hello again", user: "0x123..." },
        },
      ],
    },
  },
});
```

## API Reference

### `createTestIndexer()`

Creates a test indexer instance. This is the main entry point for all tests.

```typescript
import { createTestIndexer } from "generated";

const indexer = createTestIndexer();
```

### `indexer.process(options)`

Processes events and returns the result with all entity changes. Accepts chain-specific configuration for each of the three modes (auto-exit, block range, simulate).

```typescript
const result = await indexer.process({
  chains: {
    1: {},                                    // Auto-exit
    10: { startBlock: 100, endBlock: 200 },   // Block range
    137: { simulate: [{ ... }] },             // Simulate
  },
});
```

The returned `result.changes` array captures every entity set and delete per block, making it ideal for snapshot assertions.

### Entity Access

Read and write entities between processing runs using typed accessors on the indexer instance:

```typescript
// Read entities
const account = await indexer.Account.get("0x123...");          // Returns entity or undefined
const account = await indexer.Account.getOrThrow("0x123...");   // Throws if not found
const accounts = await indexer.Account.getAll();                // Returns all entities

// Write entities (preset state before processing)
indexer.Account.set({ id: "0x123...", balance: 100n });
```

### Chain Information

The test indexer exposes chain and contract configuration:

```typescript
const indexer = createTestIndexer();
indexer.chainIds;                    // [1, 42161]
indexer.chains[1].id;                // 1
indexer.chains[1].startBlock;        // 0
indexer.chains[1].ERC20.addresses;   // ["0x..."]
```

### Test Helpers

Additional test utilities are available from the `envio` package:

```typescript
import { TestHelpers } from "envio";
const { Addresses } = TestHelpers;

const userAddress = Addresses.defaultAddress;
```

## Assertions

The testing framework works best with Vitest's built-in assertions. The most powerful pattern is **inline snapshots**, which Vitest auto-fills on the first run:

```typescript
t.expect(result).toMatchInlineSnapshot(`...`);
```

Other common assertion patterns:

```typescript
// Check specific entity values
const user = await indexer.User.getOrThrow("0x123...");
t.expect(user.balance).toBe(100n);

// Verify entity existence
const account = await indexer.Account.get("0x123...");
t.expect(account).toBeDefined();

// Check changes count
t.expect(result.changes).toHaveLength(1);
t.expect(result.changes[0].eventsProcessed).toBe(3);
```

## Examples

### Testing Entity Creation with Simulate

```typescript
import { describe, it } from "vitest";
import { createTestIndexer, type User } from "generated";
import { TestHelpers } from "envio";
const { Addresses } = TestHelpers;

describe("Greeter indexer", () => {
  it("creates a User entity from a NewGreeting event", async (t) => {
    const indexer = createTestIndexer();
    const userAddress = Addresses.defaultAddress;
    const greeting = "Hi there";

    await indexer.process({
      chains: {
        137: {
          simulate: [
            {
              contract: "Greeter",
              event: "NewGreeting",
              params: { greeting, user: userAddress },
            },
          ],
        },
      },
    });

    const expectedUser: User = {
      id: userAddress,
      latestGreeting: greeting,
      numberOfGreetings: 1,
      greetings: [greeting],
    };

    const actualUser = await indexer.User.getOrThrow(userAddress);
    t.expect(actualUser).toEqual(expectedUser);
  });
});
```

### Testing Entity Updates with Multiple Events

```typescript
it("increments greeting count for repeat greetings", async (t) => {
  const indexer = createTestIndexer();
  const userAddress = Addresses.defaultAddress;

  await indexer.process({
    chains: {
      137: {
        simulate: [
          {
            contract: "Greeter",
            event: "NewGreeting",
            params: { greeting: "Hi there", user: userAddress },
          },
          {
            contract: "Greeter",
            event: "NewGreeting",
            params: { greeting: "Oh hello again", user: userAddress },
          },
        ],
      },
    },
  });

  const user = await indexer.User.getOrThrow(userAddress);
  t.expect(user.numberOfGreetings).toBe(2);
  t.expect(user.latestGreeting).toBe("Oh hello again");
});
```

### Testing with Preset Entity State

```typescript
it("updates an existing account balance", async (t) => {
  const indexer = createTestIndexer();

  // Preset an entity before processing
  indexer.Account.set({ id: "0x123...", balance: 500n });

  await indexer.process({
    chains: {
      1: {
        simulate: [
          {
            contract: "ERC20",
            event: "Transfer",
            params: { from: "0x123...", to: "0x456...", value: 100n },
          },
        ],
      },
    },
  });

  const sender = await indexer.Account.getOrThrow("0x123...");
  t.expect(sender.balance).toBe(400n);
});
```

### Snapshot Testing with Real Chain Data

```typescript
it("captures the first mint event via inline snapshot", async (t) => {
  const indexer = createTestIndexer();

  const result = await indexer.process({
    chains: {
      1: {
        startBlock: 10_861_674,
        endBlock: 10_861_674,
      },
    },
  });

  // Vitest auto-fills this on the first run — review and commit
  t.expect(result).toMatchInlineSnapshot();
});
```

## Troubleshooting

### Common Issues

- **"Cannot find module 'generated'"**: Run `pnpm envio codegen` to generate the testing library.

- **Tests timing out**: If using real chain data (auto-exit or block range), ensure `ENVIO_API_TOKEN` is set and network connectivity is available.

- **"Entity not found"**: Use `indexer.Entity.get()` (returns `undefined`) instead of `getOrThrow()` if the entity may not exist, or verify your handler logic creates the expected entities.

- **Snapshot mismatches**: After intentional changes to handler logic, run `pnpm vitest run --update` to update inline snapshots.

If you encounter any issues, reach out on [Discord](https://discord.gg/Q9qt8gZ2fX).

## Migrating from MockDb

:::warning Deprecated
The `MockDb` testing API has been removed in favor of `createTestIndexer()`. See the [migration guide](/docs/HyperIndex/migrate-to-v3#deprecated-mockdb-testing-api) for a complete diff and cheat sheet.
:::

| Old (`MockDb`) | New (`createTestIndexer`) |
|---|---|
| `MockDb.createMockDb()` | `createTestIndexer()` |
| `Contract.Event.createMockEvent({...})` | Inline in `simulate: [{ contract, event, params }]` |
| `Contract.Event.processEvent({event, mockDb})` | `indexer.process({ chains: { id: { simulate } } })` |
| `mockDb.entities.Entity.get(id)` | `await indexer.Entity.getOrThrow(id)` |
| `mockDb.entities.Entity.set({...})` | `indexer.Entity.set({...})` |
| Manual handler threading & event chaining | Automatic — pass multiple events in the `simulate` array |
